## **What `idle_in_transaction_session_timeout` Does**

**Definition:**
It’s a PostgreSQL setting (milliseconds) that **automatically kills a connection** if it sits *inside a transaction* without running any queries for longer than the timeout.

* **"Idle in transaction"** = You’ve done `BEGIN` (or started an implicit transaction), but haven’t sent another SQL command yet — the transaction is still open, the server is holding locks, and resources are pinned.
* If the timer expires → PostgreSQL terminates the connection with

  ```
  terminating connection due to idle-in-transaction timeout
  ```
* This is different from `statement_timeout` (which only counts while a query is running) and `idle_session_timeout` (which kills idle sessions even outside a transaction).

**Why it exists:**

* To prevent forgotten open transactions from blocking other queries and holding locks forever.
* To free up resources if an app hangs mid-transaction.

**Why it bit you:**

* Your insert pipeline sometimes opens a transaction (in `process_block`) and waits while processing data or fetching RPC results.
* If that gap > 30 s (your old setting), Postgres killed the connection → `AsyncConnection [BAD]`.

# 2) Bash script to apply the SQL and dump enums

Save as `post_migration_setup.sh` and run `bash post_migration_setup.sh`.

```bash
#!/usr/bin/env bash
set -euo pipefail

# --- Edit these or export as env vars beforehand ---
PGHOST="${PG_HOST:-127.0.0.1}"
PGPORT="${PG_PORT:-5432}"
PGDATABASE="${PG_DB:-postgres}"
PGUSER="${PG_USER:-pgadmin}"          # superuser or role with ALTER ROLE priv
PGAPPUSER="${PG_APP_USER:-blockchain}"  # your application role
# If you need a password, export PGPASSWORD in the environment

# --- Apply role-level timeout settings ---
psql "host=$PGHOST port=$PGPORT dbname=$PGDATABASE user=$PGUSER" -v ON_ERROR_STOP=1 <<SQL
ALTER ROLE ${PGAPPUSER} SET idle_in_transaction_session_timeout = 0;
-- Optional:
-- ALTER ROLE ${PGAPPUSER} SET statement_timeout = 0;

-- Show effective settings
SELECT rolname, rolconfig FROM pg_roles WHERE rolname = '${PGAPPUSER}';
SQL

# --- Dump schema DDL (includes enums) ---
# Change --schema if your enums live in a specific schema (e.g., blockchain)
# If they’re spread across schemas, you can omit --schema to dump all DDL
SCHEMA_TO_DUMP="${PG_SCHEMA:-blockchain}"
OUTFILE="${ENUMS_DUMP_FILE:-enums_backup.sql}"

echo "Dumping schema DDL (includes enums) for schema '${SCHEMA_TO_DUMP}' to ${OUTFILE} ..."
pg_dump \
  --schema-only \
  --schema="${SCHEMA_TO_DUMP}" \
  --file="${OUTFILE}" \
  "host=$PGHOST port=$PGPORT dbname=$PGDATABASE user=$PGUSER"

echo "Done. Restore later with:"
echo "  psql \"host=$PGHOST port=$PGPORT dbname=$PGDATABASE user=$PGUSER\" -f ${OUTFILE}"
```

# 3) Quick enum verification query (optional)

```sql
SELECT t.typnamespace::regnamespace AS schema,
       t.typname                      AS enum_type,
       e.enumlabel                    AS label,
       e.enumsortorder
FROM pg_enum e
JOIN pg_type t ON t.oid = e.enumtypid
WHERE t.typnamespace::regnamespace::text IN ('blockchain')   -- adjust if needed
ORDER BY schema, enum_type, e.enumsortorder;
```

## **1. Bitcoin Core RPC tuning**

Edit your `bitcoin.conf` (usually `~/.bitcoin/bitcoin.conf`) and add/adjust:

```
# --- RPC performance ---
rpcthreads=160        # Thread pool for handling RPC calls
rpcworkqueue=1024     # Max queued requests before refusing
rpcclienttimeout=300  # Seconds before RPC times out (match Python timeouts)
server=1              # Ensure RPC server mode is on
```

Then restart Bitcoin Core:

```bash
bitcoin-cli stop
bitcoind -daemon
```

Check it took effect:

```bash
bitcoin-cli getnetworkinfo | jq '.connections_out'
bitcoin-cli getrpcinfo
```

## **2. PostgreSQL connection tuning**
